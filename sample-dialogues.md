## 样例 2：条文教学示例

**Agent**

```
### 第6条 · 质量源于设计
💡 先想一个场景
> 你是一个 CRA，第一次去一个新中心做监查，发现这个中心的知情同意书签署流程经常漏掉「给受试者副本」这一步。你向 PM 反映，PM 说：「这有什么，后面补一下就行了。」——这个回复有什么问题？

📜 条文原文
> 临床试验的设计和实施应当纳入质量源于设计的方法，识别试验的关键质量因素及相关风险，并采用与之相称的风险控制措施，保护试验参与者的权益和安全，确保结果可靠。

🔑 这条在说什么
质量不是在试验结束后靠稽查「查出来」的，而是在方案设计阶段就要「设计进去」的。识别关键质量因素（比如 ICF 签署流程），预设风险控制措施（比如标准化 checklist），让问题在发生前就被防住。

🔄 和旧版（2020）有什么不同
- 2020：质量管理体系的笼统要求，没有 QbD 概念
- 2026：首次将 ICH E6(R3) 的 QbD 理念写入总则，是 2026 版最重要的理念变化

👤 对你的影响
- 📋 **CRA**（你的角色）：你发现 ICF 漏给副本，这不只是「流程不规范」，而是关键质量因素的失控。你应该建议 PM 在方案设计层面加 checklist 机制，而非事后补救。监查计划本身也要体现 QbD——识别高风险中心、预设监查频率和深度。

⚠️ 容易踩的坑
> 「质量靠稽查就能保证」——稽查是事后检查，QbD 强调事前设计。两者互补，但 QbD 在前。

---
📍 当前位置：第一章「总则」→ 第6条「质量源于设计」
📊 整体进度：6/54条 · ⏱ 约8分钟

有疑问吗？没有的话回复「继续」，有想问的随时说。
```

## 样例 3：章节测验前询问

**Agent**

```
这一章还有疑问吗？没有的话我们做个小测验（3道题）。
```

**用户**

```
我有问题：第14条说的跟踪审查≤12个月，是所有试验都这样吗？低风险的也12个月审一次？
```

**Agent**

```
好问题。第14条原文说的是「审查的频率应当根据试验的风险程度而定，时间间隔不超过12个月」。意思是：
- 高风险试验：可能需要每3-6个月审一次
- 低风险试验：最长可以12个月审一次
- 但不允许超过12个月

所以「不超过12个月」是最低要求，伦理审查委员会可以根据风险程度设更短的间隔。

还有其他问题吗？没有的话我们做测验。
```

先快速过一下术语的变化。2026版更新了四个核心术语：

- 受试者 → 试验参与者（强调主动角色，知情同意是参与者行使自主权的过程）

### 风险相称管理原则
🔑 核心变化
2026 版多处引入「风险相称」原则：监查的范围和程度应与试验的复杂性和风险相称（第40条），稽查方式应与风险相称（第41条），质量控制应采用基于风险的方法（第41条）。

🔄 和旧版的不同
2020 版第 31 条提到「基于风险的质量管理」，但未充分展开。2026 版将风险相称原则贯穿监查、稽查、质量控制三大环节。

👤 对你（PI）的影响
PI 在制定监查计划时应根据试验风险程度确定监查频率和深度，低风险试验不必过度监查。
